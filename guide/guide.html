    Front-end Standardization Guide

    본 문서는 프로젝트의 일관된 품질과 유지보수 효율성을 위해 작성된 퍼블리싱/개발 표준 가이드입니다.

    본 CSS 및 JS는 기본 가이드를 제시하기 위한 예시이며, 기획과 디자인의 요구를 구현할 때 제약 없이 수정, 확장 및 재구성하여 사용할 수 있습니다.

    단, 파일 구조와 일부 문법에는 제약이 있으므로 세부 사항은 본문 내용을 참고하시기 바랍니다.

    브라우저 호환성 및 환경

    Target Browser: Chrome 78+
    최신 문법 사용 시 해당 버전 지원 여부를 반드시 확인해야 합니다.

    웹접근성 준수

    • 본 프로젝트의 모든 웹 콘텐츠는 「웹 접근성 지침(KWCAG)」을 준수하여 제작한다.
    • 퍼블리싱 단계에서 시맨틱 구조, 키보드 접근성, 대체 텍스트, 명도 대비 등을 필수적으로 검토한다.

    디렉토리 구조 및 배포 정책

    디렉토리는 배포 주체와 역할을 기준으로 엄격히 분리하여 운영합니다.

    Root ├── _dev/ # [개발 전달용 html 폴더, 배포 대상 아님] │ ├── sub1/ # 카테고리 등 필요시 폴더 구조 │ └── my/myinfo.html │ ├── static/ # [정적 리소스 폴더, 퍼블리셔가 배포할 대상] │ ├── styles/ # 필요 시 서브 폴더 구성 │ ├── scripts/ │ ├── images/ │ └── html/contact.html # 개발 연동 없는 정적 html 파일 │ ├── src/ # [필요 시 구성하며 배포 대상 아님] │ └── styles/ # SCSS 원본 파일 │ ├── guide/ # [퍼블 가이드용 파일, 배포 대상 아님] │ ├── filelist.html # 퍼블리스트 등을 자유롭게 작성 합니다. │ └── images, styles, js ...

    구조 원칙

    • _dev 폴더(필수): 개발자에게 전달하는 html 파일이 위치합니다.
    • static 폴더(필수): 정적 리소스 파일들이 위치하며 추후 퍼블리셔가 배포 할 대상입니다.
    • src 폴더(선택): 필요시 구성하며 static 폴더로 컴파일하여 배포합니다.
    • guide 폴더(필요): 퍼블리스트와 퍼블 가이드 등을 위한 파일들이 위치합니다.
    • 그 외 필요 시 하위 폴더 생성 가능

    HTML 가이드라인

    리소스 경로

    실제 프로젝트에서는 리소스는 절대경로로 변경해 주세요. 가이드는 로컬에서 볼 수 있도록 상대경로로 제작했습니다.

    인클루드

    반복되는 공통 레이아웃(header, aside, footer 등) 요소는 fetch 또는 innerHTML 등 js로 구현 (node 또는 서버 언어 X)

    CSS 가이드라인

    Variables

    reset.css 혹은 공통 CSS 파일 내 :root 영역에 주요 컬러를 변수로 정의합니다.

    CSS 파일 분리 전략

    유지보수와 배포의 용이성을 위해 CSS는 공통 영역과 각 메뉴별 영역으로 분리합니다.

    • Common: common.css, layout.css (헤더, 푸터, 공통 컴포넌트)
    • Page Specific: 1 Depth 메뉴(또는 카테고리) 기준으로 CSS 파일을 생성합니다.
    /* 파일 생성 예시 */
    common.css /* 공통 */
    main.css /* 메인 페이지 */
    product.css /* 쇼핑/상품 관련 (1 Depth) */
    myroom.css /* 마이페이지 관련 (1 Depth) */
    event_newyear.css /* 새해 이벤트 (1 페이지 용) */

    주의사항

    has() 가상 클래스 선택자 사용 불가 (크롬 78 버전 미지원)

    디자인 위한 유틸리티 선택자 지양 (.mt10, .point-color, .bold, .d-flex, align-center 등)

    important 지양

    JavaScript 및 공통 요소 처리

    Optional Chaining (?.) 연산자 사용 불가 (크롬 78 버전 미지원)

    JS 모듈화 전략

    • Common 용 JS: 사이트 전역에서 사용되는 유틸리티 함수 모음.
    • 특정 타겟 페이지 용 JS: 유지보수와 배포의 용이성을 위해 특정 페이지나 메뉴에서만 사용되는 기능은 별도 JS 파일로 분리합니다.

    UI 및 자산(Assets) 관리

    반응형 브레이크 포인트

    반응형 작업을 위한 Break Point는 임의로 설정하지 않으며, 기획 및 디자인 팀과 협의된 수치를 사용합니다.

    이미지 최적화 가이드

    고해상도 디스플레이 대응 및 로딩 속도 최적화를 위해 아래 규칙을 준수합니다.

    항목 규격 / 설정값 비고
    JPEG 압축률 70% 사진, 그라데이션이 많은 이미지
    PNG 압축률 80% 투명 배경, 단순한 그래픽
